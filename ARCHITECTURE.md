# Cosilium-LLM: Архитектура мульти-агентного аналитического интеллекта

> **Версия:** 1.0
> **Дата:** 2026-02-01

## Назначение системы

Мульти-агентный аналитический интеллект высшего уровня для:
- бизнес-стратегии
- ресёрча
- инвестиций
- разработки
- технического и методологического аудита

**Целевой стандарт:** результаты стабильно превосходят уровень сильных человеческих экспертов, аналитических отделов и инвестиционных комитетов.

## 2. Итеративный процесс анализа (2–3 итерации)

### Итерация 1 — Независимый анализ
Каждый агент:
- проводит самостоятельный анализ задачи
- формирует первичные выводы
- явно фиксирует:
  - предпосылки
  - методологию
  - используемые количественные подходы
  - степень уверенности

### Итерация 2 — Взаимная критика (adversarial mode)
Агенты обмениваются результатами и критикуют по критериям:

| Критерий | Описание |
|----------|----------|
| Логическая корректность | Нет ли разрывов в цепочке рассуждений |
| Явные и скрытые предпосылки | Все ли допущения озвучены |
| Качество данных | Релевантность и достоверность |
| Методология | Адекватность выбранных методов |
| Альтернативные объяснения | Рассмотрены ли другие гипотезы |
| Границы применимости | Где выводы перестают работать |
| Ясность понятий | Однозначность терминов |
| Когнитивные искажения | Confirmation bias и др. |
| Фальсифицируемость | Можно ли опровергнуть вывод |
| Математическая корректность | Правильность вычислений |

Каждый агент обязан:
- выявить слабые места
- указать математические или методологические ошибки
- предложить способы количественного усиления

### Итерация 3 — Совместная синтезация
Агенты:
- устраняют слабости
- согласуют противоречия
- усиливают устойчивость выводов
- формируют финальные формализованные решения

## 3. Приоритет математического и количественного анализа

### 3.1 Общий принцип
```
Если можно посчитать — нужно посчитать.
Если нельзя посчитать — нужно объяснить почему.
Если нельзя фальсифицировать — вывод считается слабым.
```

| Утверждение | Требование |
|-------------|------------|
| Можно выразить формулой | → Должно быть выражено формулой |
| Можно оценить численно | → Должно быть оценено численно |
| Нельзя формализовать | → Явно указать причину |

### 3.2 Обязательные типы количественного анализа

- Вероятностные модели
- Математическое ожидание, дисперсия, доверительные интервалы
- Сценарный анализ
- Sensitivity analysis
- Risk-adjusted оценки
- Cost–benefit модели
- Формальные ограничения и допущения

### 3.3 Проверка точности
Каждое ключевое численное утверждение должно:
- иметь источник или логическое обоснование
- быть проверено другим агентом
- сопровождаться условиями, при которых оно перестаёт быть верным

---

## 4. Работа с внешним поиском

Используется **ТОЛЬКО ЕСЛИ:**
- обнаружена нехватка данных
- возник спорный или критически важный факт
- есть прямое указание пользователя

> Поиск используется строго для верификации данных, а не подмены анализа.

---

## 5. Управление рассуждениями

- Внутренние рассуждения и диалоги **не показываются** пользователю
- Весь межагентный диалог:
  - структурируется
  - сохраняется в БД
  - доступен пользователю по запросу

## 6. RAG-агенты

### 6.1 RAG: Оптимальные промты агентов
- Самоформирование промтов
- Самоанализ
- Самоулучшение

### 6.2 RAG: Образы мышления величайших умов
- Минимум — не хуже человеческого образца
- Целевой уровень — лучше

### 6.3 RAG: Контекст текущей задачи
- Используется только по указанию пользователя
- Имеет **высший приоритет**

---

## 7. Иерархия источников знаний

```
1. RAG текущей задачи          (высший приоритет)
2. Верифицированный внешний поиск
3. RAG образов мышления
4. RAG промтов
5. Внутренние знания моделей   (низший приоритет)
```

- С альтернативными сценариями
- С ограничениями применимости
- С количественными обоснованиями

|-------|-------------|-------|----------------------|---------------------|
| ... | ... | ... | ... | ... |

### C. Формализованный итог
Финальный результат обязан включать:
- Одну или несколько итоговых формул
- Расшифровку всех переменных
- Диапазоны допустимых значений
- Интерпретацию на человеческом языке

**Пример:**
```
ROI = (G - C) / C × 100%

где:
  ROI — возврат инвестиций (%)
  G   — полученная выгода ($)
  C   — затраты ($)

Диапазон: ROI ∈ [-100%, +∞)
Интерпретация: ROI > 0 означает прибыльность инвестиции
```

### D. Итоговые рекомендации
- Несколько лучших вариантов решений
- Trade-offs между ними
- Где каждый вариант оптимален
- Где каждый вариант опасен
- Количественное сравнение вариантов

## 9. Критерий зрелости системы

Система считается зрелой, если:
- [ ] Качество выводов улучшается от задачи к задаче
- [ ] Промты агентов эволюционируют
- [ ] Математическая строгость растёт
- [ ] Выводы стабильно превосходят экспертные человеческие решения

---

## 10. Финальный принцип

> **Цель системы — не дать ответ, а создать лучшее возможное решение.**

- Если уверенность невозможна — это должно быть явно указано
- Если вывод нефальсифицируем — он считается слабым
- Если можно формализовать — это обязательно
